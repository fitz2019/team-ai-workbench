# GitHub Ops

Use when a team wants AI-assisted GitHub repository operations beyond local git commands.

## Use Cases

- issue triage
- PR review status tracking
- CI failure inspection
- release checklist generation
- stale item follow-up

## Requirements

- `gh` CLI configured
- repository access available

## Warning

This is optional team operations capability, not a coding baseline.
It should be enabled only when the team actually wants GitHub workflow automation in its AI operating model.
