# Backend Role

Production-ready role pack for Go backend repositories.

Includes:

- backend-specific `.agents` modules
- backend example index and focused implementation examples
- backend Codex role overlays
- Go/backend skills
- Go-backend test-driven-development skill
- ECC-derived backend reviewer and build resolver roles

Use this role when the target repository contains backend services, APIs, workers, MQ consumers, DB migrations, or Redis/cache logic.
