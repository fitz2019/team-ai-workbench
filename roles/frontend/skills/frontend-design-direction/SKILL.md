# Frontend Design Direction

Use when the task is not only to make UI function, but to make it feel intentional and product-appropriate.

## Use Cases

- visual polish
- interface refinement
- layout hierarchy decisions
- non-generic product-facing pages

## Guidance

- choose an explicit design direction before coding
- match layout density and visual tone to the domain
- avoid generic generated patterns that do not fit the product
- verify text fit and responsive stability
