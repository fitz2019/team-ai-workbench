# Frontend Accessibility

Use when building or reviewing interactive UI, forms, dialogs, navigation, or any interface that must support keyboard and assistive technology users.

## What to Check

- labels connected to controls
- semantic elements used for interactive behavior
- keyboard reachability
- focus visibility and order
- alt text and landmark structure
- ARIA only where native semantics are insufficient

## Rule

Do not call a frontend task complete if the UI works only with a mouse.
