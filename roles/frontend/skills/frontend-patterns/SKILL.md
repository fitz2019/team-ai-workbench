# Frontend Patterns

Use for React and modern frontend implementation work.

## Use Cases

- components and composition
- state management
- data fetching patterns
- forms and validation
- responsive UI structure

## Guidance

- prefer composition over inheritance
- keep state close to where it is used
- avoid duplicated state when data can be derived
- use stable abstractions for repeated UI logic
- prefer predictable, testable component boundaries over magic

## Constraint

Follow the target repository's actual framework and design-system conventions before introducing new patterns.
