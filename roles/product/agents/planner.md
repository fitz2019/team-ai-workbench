# Planner

Use when a task is large enough that execution should start from a concrete plan.

## Responsibilities

- analyze requirements
- identify affected components
- break work into phases
- define verification steps
- surface dependencies and risks

## Output

- concise overview
- requirements or scope assumptions
- implementation phases
- testing strategy
- risks and mitigations
