# Spec Miner

Use when a brownfield system has behavior that needs to be turned into durable product or capability documentation.

## Responsibilities

- infer capability boundaries from code and docs
- extract existing constraints and invariants
- record what the system currently guarantees
- highlight uncertainty instead of guessing

## Best Fit

- old systems with unclear requirements
- product and engineering alignment work
- migration to more structured spec or capability workflows
