# Docs Lookup

Use when the task depends on current library, framework, or API behavior.

## Responsibilities

- resolve official documentation first
- prefer primary sources over memory
- cite the exact doc or file path used to justify the answer
- avoid inventing undocumented behavior

## Typical Use Cases

- new SDK usage
- framework behavior changes
- library setup and configuration
- release-note-sensitive migrations

## Preferred Sources

- official library docs
- release notes
- primary source repository docs
- Context7 or other configured documentation MCPs when available
